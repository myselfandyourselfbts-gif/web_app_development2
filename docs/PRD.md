# PRD文件: 讀書筆記本 (Reading Notebook)

## 1. 專案概述
- **背景與動機**：現代人閱讀量大，但常會忘記書中內容或心得。本系統旨在提供一個簡單易用的平台，讓使用者能快速記錄讀過的書籍資訊與心得。
- **目標用戶**：熱愛閱讀的學生、上班族，以及希望建立個人知識庫的讀者。
- **核心價值主張**：提供輕量化、專注於書籍記錄與檢索的筆記系統，不再讓閱讀記憶輕易流失。

## 2. 功能需求
- **新增書籍筆記**
  - 作為讀者，我希望能夠新增書名、心得與評分，以便記錄我的閱讀體驗。
- **檢視與列表書籍**
  - 作為讀者，我希望能夠在首頁看到我所有的讀書筆記列表，以便回顧閱讀歷程。
- **關鍵字搜尋功能**
  - 作為讀者，我希望能夠使用關鍵字搜尋書名或心得，以便快速找回特定的筆記。
- **編輯書籍筆記**
  - 作為讀者，我希望能夠修改已建立的筆記內容與評分，以便補充或修正心得。
- **刪除書籍筆記**
  - 作為讀者，我希望能夠刪除不需要的筆記，以便維持清單的整潔。

## 3. 非功能需求
- **技術限制**：系統使用 Flask 作為後端框架，Jinja2 進行前端模板渲染，以及 SQLite 作為輕量級的關聯式資料庫儲存筆記。
- **效能與安全考量**：
  - 系統響應時間應在 1 秒內。
  - 對於使用者輸入的欄位（如書名、心得、搜尋關鍵字）需進行基礎的防護以避免 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍
- **Must Have (必須有)**：新增筆記（書名、心得、評分）、筆記列表展示、關鍵字搜尋功能。
- **Should Have (應該有)**：編輯與刪除筆記功能。
- **Nice to Have (加分項)**：筆記分類/標籤、閱讀進度標記。
